# Security Policy 🛡️

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 1.x     | ✅ Fully supported |

## Reporting a Vulnerability

**Please do not open public GitHub issues for security vulnerabilities.**

Instead, send a report to **security@greenhat.io** (placeholder — update before launch).

### What to include

- Contract address and network
- Description of the vulnerability
- Steps to reproduce (if applicable)
- Proposed fix (optional)

### Response timeline

- **24 hours**: Acknowledgment of receipt
- **7 days**: Initial assessment and mitigation plan
- **30 days**: Fix deployed (depending on severity)

## Responsible Disclosure

We follow a 90-day responsible disclosure window. After a fix is deployed, we will:

1. Deploy the fix to mainnet
2. Publish a post-mortem
3. Credit the reporter (if desired)

## Audits

This project has been audited with:

- **Slither** — Static analysis (run on every PR)
- **Solhint** — Code style & best practices

A formal third-party audit is recommended before mainnet launch.
